Parameters are input values you can pass to your report to filter data or customize output.

💡 Examples & Tips

Date Filter: Name: startDate, Type: Date, Default: new java.util.Date()

Text Filter: Name: customerName, Type: String, Default: "All"

Number Filter: Name: minAmount, Type: BigDecimal, Default: 0

Use in SQL: WHERE date >= $P{startDate} AND amount >= $P{minAmount}

Tip: Parameters must be passed when generating the report. Use them in your SQL query to filter data.

Variables allow you to perform calculations, store values, and create running totals.

💡 Examples & Tips

Running Total: Name: totalSales, Type: BigDecimal, Calculation: Sum, Expression: $F{price}

Record Counter: Name: recordCount, Type: Integer, Calculation: Count, Expression: $F{id}

Page Numbering: Use built-in variable $V{PAGE_NUMBER}

Tip: Variables are evaluated in the order they're declared. One variable can reference another declared before it.

Datasets allow you to organize multiple queries within one report, useful for charts, tables, and aggregations.

💡 Examples & Use Cases

Sales by Category: Name: CategorySales, Query: SELECT category, SUM(amount) FROM sales GROUP BY category

Top Products: Name: TopProducts, Query: SELECT product_name, quantity FROM orders ORDER BY quantity DESC LIMIT 10

Monthly Trends: Name: MonthlyData, Query: SELECT MONTH(date), AVG(value) FROM metrics GROUP BY MONTH(date)

Use Case: Create charts, crosstabs, or tables that use different data than your main report query.

Tip: Datasets can accept parameters from the main report. Use $P{parameterName} in dataset queries.

Subreports embed one report inside another, perfect for combining data from multiple datasources.

💡 Examples & Use Cases

Customer Orders: Main report shows customers, subreport shows their orders (different datasource)

Multi-Source Invoice: Main report from billing DB, subreport items from inventory DB

Summary with Details: Main report shows totals, subreport lists all transactions

Use Case: Combine data from different databases or tables that can't be joined with SQL.

Tip: You can pass parameters from main report to subreport, and each subreport can use a different datasource!
